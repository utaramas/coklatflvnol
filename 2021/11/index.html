Pengalaman Belajar Github Secara Otodidak

Tinggalkan Balasan

Puih, lelahnya belajar github sendirian. Kali ini mimin mau share pengalaman belajar Github secara otodidak. Sekitar sebulan terakhir mimin utak-atik localhost, github, vs code, wamp server, youtube, ccleaner, docker compose, wordpress, git bash, ubuntu. Cukup menantang juga sih.

Postingan ini juga untuk menambah commit baru di github ya 🙂

Latar belakang belajar github

Alkisah mimin sedang mencari hosting gratis buat wordpress selain 000webhost. Browsing-browsing, ketemu kalau github bisa untuk tempat hosting wp kita walaupun sifatnya statis. Untuk pengolahannya sendiri memerlukan yang namanya server di localhost alias komputer/laptop kita sendiri.

Mulailah mimin mengunduh WAMP. Proses instalasi wp nya sendiri tidak dimulai dari awal sebab mimin punya ide untuk mengupload hasil backup wp terdahulu. Ternyata bisa juga localhost untuk tempat backup.

Mimin perlu melakukan instalasi ulang ioncube di localhost dan tidak beberapa lama maka tayanglah wordpress dari hasil backup 🙂

Masalah SSL Sertifikat

Masalah pertama muncul adalah setelah melakukan push & commit ke git, css tidak muncul termasuk gambar. Melihat solusi saat itu koq terasa berat ya buat menyelesaikannya. Alhasil cara memasang ssl sertifikat mimin pending dulu.

Lucu juga sih, kenapa localhost perlu ssl sertifikat.

Docker-Compose Up yang menggoda

Muncul godaan lain, yaitu untuk menginstall wordpress menggunakan docker-compose.yml, tutorial aslinya pun tidak begitu jelas. Perangkat yang dibutuhkan adalah Docker untuk windows, wsl2, ubuntu dan visual studio code. Semua perangkat tersebut harus saling terkoneksi.

Di Visual Studio juga memerlukan lain ekstensi Docker. Masalah utama mimin adalah bagaimana melakukan docker-compose up di direktori /var/www/html. HIngga artikel ini ditulis masalah ini belum rampung 100%.

Akun Github Tersuspend

Mungkin akibat terlalu sering melakukan perintah git add, git commit dan git push, akhirnya akun github mimin tersuspen. Tahukan berapa banyak koding wordpress? mimin pernah melakukan upload ke repositori satu per satu. Mungkin dikira spam ya 🙁

Cuma proses suspen tidak langsung terjadi saat itu. Bisa jadi membutuhkan waktu sekitar 1 atau 2 pekan kemudian.

Cuma kalau kamu mengalami akun git yang tersuspen atau kalau kamu ingin memiliki banyak akun git, dari 1 email saja kamu sudah bisa membuat banyak akun git.

Kamu cukup melakukan permutasi alamat email kamu. Contohnya alamatemailsaya@gmail.com bisa dimodifikasi menjadi a.lamatemailsaya@gmail.com, kamu lihat perbedaannya?

Nanti verifikasi yang masuk akan tetap ke alamat email utama koq. Kamu cukup mengganti username git aja. Nah omong-omong username, kalau menurut aliran Piranhamas, username juga penting buat SEO loh.

Sebab mimin mencoba untuk mempraktekan teori Piranhamas menggunakan Github.

Pentingnya sign out dari aplikasi Git

Walaupun akun kamu tersuspen, kamu tetap bisa login ke github. Masalahnya, github tidak otomatis mengeluarkan status login kamu di aplikasi seperti visual studio, git bash dan github desktop.

Jadi ketika kamu sudah pusing-pusing melakukan git add, git commit dan git push, pembaruan yang kamu lakukan tidak akan masuk kedalam repositori. Sayang bukan?

Banyak berterima kasih kepada Youtuber dari India

Kalau kamu menemui kendala, salah satu cara mencari solusi adalah mencari tutorial di YT. Mimin banyak mendapat solusi dari Youtuber asal negeri India. Tentunya kata kunci yang kamu ketik harus berbahasa Inggris ya dan kamu harus mengerti sedikit bahasa Inggris hehe.

Banyak juga sih selain dari India, ada dari native dan juga ada yang dari negeri kita sendiri, misalnya channel webpromamingunpas.

Pentingnya SSL, status WAMP hijau, kecepatan loading localhost

Akhirnya mimin bisa menyelesaikan masalah SSL, walaupun ada plugin yang kurang support sehingga harus dinonaktifkan.

Penting juga agar WAMP Server kamu berwarna hijau, artinya semua service aktif. Sebab kalau tidak akan banyak timbul masalah ke depan. Alih-alih kamu sudah membuat banyak project, sebaiknya status hijau harus sudah dimulai dari awal.

Memperbaiki kecepatan loading adalah tantangan kamu selanjutnya. Lagi-lagi, Youtube dan Google adalah teman terbaik kamu.

Perlunya komunitas pengguna Git di tanah air

Mimin belum tahu ada atau tidaknya komunitas pengguna Git di Indonesia. Paling tidak kalau kamu menemukan masalah ada tempat buat bertanya dan berdiskusi.

Sebab memang butuh kesabaran juga sih dalam menggunakan Git. Apalagi kalau kamu memang memiliki passion koding atau marketing.

Marketing disini adalah menggunakan git sebagai salah jet dunia maya yang bisa kita manfaatkan untuk menawarkan produk/jasa kita.

Fokus mimin selanjutnya adalah Netlify, sebab untuk subdomain yang didapat memiliki DA yang lebih besar dari Github. Bisa jadi kecepatan index akan lebih cepat di Netlify, begitu juga mungkin dengan kualitas backlink yang didapat.

Alternatif solusi hosting gratis untuk WordPress

Seperti yang disampaikan diawal artikel, github bisa menjadi alternatif solusi untuk blog WordPress kamu secara murah/gratis, tanpa perlu takut kehabisan kuota space dan bandwidth.

Berbeda dengan layanan 000webhost atau byethost yang memiliki keterbatasan kalau kamu sudah mencapat limit tertentu.

Untuk kecepatan index Google sendiri, yang mimin rasakan 000webhost agak lambat. Untuk submit sitemap sendiri agak sulit di Google Search Console.

Bisa Custom Domain di Git

Jadi kalau kamu tidak mau alamat blog kamu memiliki subdomain seperti https://utaramas.github.io/coklatflvnol/

kamu bisa melakukan custom domain url diatas dengan nama domain yang kamu inginkan. Jadi alurnya localhost -> Github -> Custom Domain. Hosting aslinya berada di komputer kita, Git menerima file statis, kemudian di redirect ke alamat Custom Domain.

Mimin sendiri belum pernah mencobanya. Jadi kalau kamu punya nama domain dengan DA PA yang besar, nilai Spam Score yang kecil, kamu bisa manfaatkan nama domain tersebut.

Titik start yang berbeda

Mungkin mimin termasuk yang belakangan mengenal localhost, padahal sudah mengenal istilah tersebut dari mulai tahun 2000an awal. Ya mimin tidak pernah menginstall WordPress di localhost sebab salah satu teman memberi tahu kalau ada penyedia hosting gratis. Waktu itu mimin menggunakan Byethost.

Selanjutnya mimin baru mulai membeli nama domain dan layanan hosting berbayar seperti di Dewaweb. Itupun setelah mengikuti Meetup WordPress Jakarta. Nah mimin belum cek lagi apakah Dewaweb memiliki fasilitas koneksi ke Git.

Dalam perjalanan kesini, mimin lebih banyak memanfaatkan website yang bisa dimanfaatkan secara free. Untuk supaya orang mengenal produk/jasa/profil kita sepertinya tidak memerlukan fitur premium berbayar.

Hanya cukup memperdalam bagaimana cara kerja mesin sebuah website.

Banyak yang bisa dikembangkan lagi dari Git

Menurut perkiraan mimin banyak hal yang bisa dikembangkan lagi dari Git. Yang terpikir saat ini oleh mimin masih seputar digital marketing, Search Engine Optimization, backlink off-page dan alat indexing.

Mimin memang memiliki minat yang cukup besar dalam dunia blogging, seo, internet marketing dengan menggandol jet dunia maya.

WordPress sendiri masih merupakan mesin cms yang paling fleksibel dimana saja. Bisa di hosting gratisan & berbayar, Google Cloud, Amazon Web Service dan mungkin di Microsoft Azure.

Saat ini yang sedang mimin geluti secara otodidak, bagaimana melakukan optimasi WordPress di Github. Sampai bertemu pada artikel selanjutnya.
